# Contributing

The authoritative contributor guide lives at the repository root:

**→ [CONTRIBUTING.md](../CONTRIBUTING.md)**

It covers:

- The Golden Rule (no functional code without a characterization test)
- How to contribute (bugs, features, docs, refactors)
- **AI-assisted PR policy** — Codex / Claude Code / Cursor / Copilot welcome
- Dev setup, fast inner loop, running hooks and the MCP server standalone
- Test layout and narrow test runs
- Coding conventions (TypeScript strict, pure engines, no suppressions)
- Release process via [`np`](https://github.com/sindresorhus/np)
- Pull request checklist
- Bug reports and security vulnerability reporting
